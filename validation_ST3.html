Reflection on Validation Process

During the development of my CLIMA pages, I used the W3C Markup Validation Service to ensure all HTML files were error-free and followed proper standards. Initially, the validator highlighted several issues, particularly within my feedback form and content page. One key issue was related to form inputs where required attributes and proper structure were missing or inconsistent. I resolved this by ensuring all form fields had correct labels, required attributes, and valid input types, improving both validation and accessibility. Another issue involved incorrect HTML structure, including minor nesting errors and inconsistent use of elements. I carefully reviewed and corrected these to ensure all elements were properly structured and semantically correct. I also encountered issues with attributes such as empty or invalid values (e.g. placeholder or image attributes), which were flagged by the validator. These were fixed by providing valid default values and ensuring all attributes followed HTML standards. Through this process, I learned that validation is essential not only for removing errors but also for improving code quality, accessibility, and maintainability. It reinforced the importance of writing clean, semantic HTML that works consistently across browsers and devices.
